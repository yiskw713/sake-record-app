# 日本酒記録アプリ 仕様書

**バージョン:** 1.0  
**作成日:** 2026年5月19日  
**対象ユーザー:** 個人（1名）

---

## 1. システム概要

飲んだ日本酒の情報を記録・管理し、統計表示とクイズ機能を提供するWebアプリケーション。フロントエンドをGitHub Pagesで、バックエンドをGoogle Apps Script（GAS）で、データストレージをGoogleスプレッドシートで構成する。

---

## 2. システム構成

```
[ユーザー]
    ↓ HTTPS
[フロントエンド: GitHub Pages]
  ├── 入力フォームページ  (index.html)
  ├── 統計ダッシュボードページ (stats.html)
  └── クイズページ (quiz.html)
    ↓ Fetch API (POST / GET)
[バックエンド: Google Apps Script]
  ├── doPost()  → データ書き込み
  └── doGet()   → データ取得（銘柄一覧 / 統計 / クイズ）
    ↓ Sheets API
[データストレージ: Googleスプレッドシート]
  ├── 2024シート
  ├── 2025シート
  ├── 2026シート
  └── ... (年ごとに自動作成)
```

### 技術スタック

| 層 | 技術 | 備考 |
|---|---|---|
| フロントエンド | HTML / CSS / JavaScript | シングルページ構成 |
| バックエンド | Google Apps Script (GAS) | Web APIとして公開 |
| データストレージ | Googleスプレッドシート | 年別シート管理 |
| ホスティング | GitHub Pages | 静的ファイル配信 |
| 認証 | Googleアカウント認証 (GAS標準) | 自分のみアクセス可 |

なお、開発はテスト駆動で行ってください。

### 費用

| 項目 | 費用 |
|---|---|
| GitHub Pages | 無料 |
| Google Apps Script | 無料 |
| Googleスプレッドシート | 無料 |
| **合計** | **0円** |

## 3. 認証

GASのWebアプリデプロイ設定を **「自分のみ（Only myself）」** に設定する。

- アクセス時にGoogleアカウントのログインが要求される
- 登録したGoogleアカウント以外はアクセス不可
- 追加の認証システムは不要

---

## 4. データ仕様

### 4-1. スプレッドシート構造

- スプレッドシート名: `日本酒記録`
- シート名: 飲んだ年（例: `2024`, `2025`, `2026`）
- シートが存在しない年のデータが追加された場合、GASが自動でシートを作成する

### 4-2. 各シートのカラム定義

| 列 | カラム名 | 型 | 必須 | 備考 |
|---|---|---|---|---|
| A | 記録ID | 文字列 | ◯ | タイムスタンプベースの自動採番 (例: `20260519143022`) |
| B | 銘柄名 | 文字列 | ◯ | |
| C | 酒蔵名 | 文字列 | ◯ | |
| D | 都道府県 | 文字列 | ◯ | |
| E | 市町村 | 文字列 | ◯ | |
| F | 飲んだ日時 | 日時 | ◯ | ISO 8601形式 (例: `2026-05-19T14:30:00`) |
| G | 感想 | 文字列 | ✗ | 任意入力 |
| H | 登録日時 | 日時 | ◯ | GASが自動付与 |

## 5. API仕様

### 5-1. 共通仕様

- ベースURL: `https://script.google.com/macros/s/{DEPLOYMENT_ID}/exec`
- 認証: Googleアカウント（GAS標準）
- レスポンス形式: JSON

### 5-2. エンドポイント一覧

#### POST /exec — 日本酒を記録する

**リクエストボディ (JSON)**

```json
{
  "action": "addRecord",
  "brand": "獺祭",
  "brewery": "旭酒造",
  "prefecture": "山口県",
  "city": "岩国市",
  "drankAt": "2026-05-19T14:30:00",
  "note": "フルーティで飲みやすい"
}
```

**レスポンス**

```json
{
  "status": "success",
  "recordId": "20260519143022"
}
```

---

#### GET /exec?action=getBrands — 銘柄一覧を取得する（類似検索用）

**レスポンス**

```json
{
  "status": "success",
  "brands": ["獺祭", "久保田", "八海山", "...]
}
```

---

#### GET /exec?action=getStats&year=2026 — 統計データを取得する

`year` パラメータ省略時は全年集計。

**レスポンス**

```json
{
  "status": "success",
  "year": "2026",
  "totalCount": 42,
  "uniqueBrands": 30,
  "uniqueBreweries": 25,
  "weeklyAverage": 1.8,
  "monthlyAverage": 7.0,
  "prefectureBreakdown": {
    "新潟県": 8,
    "山口県": 5,
    "...": 0
  },
  "monthlyBreakdown": {
    "1": 3, "2": 5, "3": 4,
    "...": 0
  }
}
```

---

#### GET /exec?action=getQuiz — クイズ用データを取得する

**レスポンス**

```json
{
  "status": "success",
  "question": {
    "type": "brandToBrewery",
    "question": "「獺祭」の酒蔵はどれ？",
    "correct": "旭酒造",
    "choices": ["旭酒造", "久保田酒造", "八海醸造", "黒龍酒造"]
  }
}
```

**クイズタイプ**

| type | 出題形式 |
|---|---|
| `brandToBrewery` | 銘柄名 → 酒蔵名 |
| `breweryToBrand` | 酒蔵名 → 銘柄名 |

## 6. 機能仕様

### 6-1. 入力フォーム機能

#### 画面構成

- 銘柄名（テキスト入力）
- 酒蔵名（テキスト入力）
- 都道府県（プルダウン、47都道府県）
- 市町村（テキスト入力）
- 飲んだ日時（日時ピッカー、デフォルト: 現在日時）
- 感想（テキストエリア、任意）
- 送信ボタン

#### 類似銘柄サジェスト機能

1. ユーザーが銘柄名を入力（2文字以上で発動）
2. フロントエンドがGASから全銘柄一覧を取得（初回のみ、以降はキャッシュ）
3. 編集距離（Levenshtein距離）を用いて類似銘柄を算出
4. 類似度上位3件をドロップダウンで候補表示
5. ユーザーが候補を選択 → 入力フォームに自動反映
6. 候補を使用しない場合はそのまま入力を続行可能

**類似判定の閾値**

| 条件 | 表示 |
|---|---|
| 編集距離 ≤ 2 | 候補として表示 |
| 完全一致 | 「登録済み」バッジを表示 |
| 候補なし | サジェストを非表示 |

#### バリデーション

| フィールド | ルール |
|---|---|
| 銘柄名 | 必須、1〜50文字 |
| 酒蔵名 | 必須、1〜50文字 |
| 都道府県 | 必須、リストから選択 |
| 市町村 | 必須、1〜50文字 |
| 飲んだ日時 | 必須、未来日時は不可 |
| 感想 | 任意、0〜500文字 |

---

### 6-2. 統計ダッシュボード機能

#### 表示項目

**サマリーカード（年別切り替え可能）**

| 項目 | 内容 |
|---|---|
| 総杯数 | 記録した総件数 |
| 銘柄数 | ユニーク銘柄数 |
| 酒蔵数 | ユニーク酒蔵数 |
| 週平均 | 杯/週 |
| 月平均 | 杯/月 |

**グラフ**

| グラフ | 種別 | 説明 |
|---|---|---|
| 月別飲酒数 | 棒グラフ | 月ごとの記録件数 |
| 都道府県別分布 | 横棒グラフ | 上位10件 |

**年別切り替え**

- 年のタブまたはセレクトで切り替え
- 「全期間」も選択可能

---

### 6-3. クイズ機能

#### ゲームフロー

```
スタート
 ↓
問題表示（4択）
 ↓
ユーザーが選択
 ↓
正解 / 不正解の表示（正解ハイライト）
 ↓
「次の問題へ」ボタン
 ↓
10問終了後 → 結果画面（スコア表示）
```

#### 問題生成ロジック

1. GASがスプレッドシートから全記録を取得
2. ランダムに1件を正解として選択
3. 同じ列（酒蔵名 or 銘柄名）から残り3件をランダム抽出して選択肢を構成
4. 4択の順番をシャッフル

#### 結果表示

- 正解数 / 10問
- 正解率（%）
- 「もう一度プレイ」ボタン

## 7. GAS実装仕様

### ファイル構成

```
Code.gs         — エントリーポイント (doGet / doPost)
SheetService.gs — スプレッドシート操作
StatsService.gs — 統計集計
QuizService.gs  — クイズデータ生成
```

### 主要関数

| 関数 | ファイル | 説明 |
|---|---|---|
| `doGet(e)` | Code.gs | GETリクエストのルーティング |
| `doPost(e)` | Code.gs | POSTリクエストのルーティング |
| `addRecord(data)` | SheetService.gs | 記録をシートに追加 |
| `getOrCreateSheet(year)` | SheetService.gs | 年別シートの取得または作成 |
| `getAllBrands()` | SheetService.gs | 全銘柄一覧を返す |
| `getStats(year)` | StatsService.gs | 統計データを集計して返す |
| `getQuizQuestion()` | QuizService.gs | クイズ問題を生成して返す |

### シート自動作成ロジック

```
1. 飲んだ日時から年を抽出
2. その年のシートが存在するか確認
3. 存在しない場合 → 新規シートを作成し、ヘッダー行を追加
4. 存在する場合 → そのまま使用
```

### CORSヘッダー

GitHub Pagesからのクロスオリジンリクエストを許可するため、レスポンスに以下を付与する。

```javascript
.setHeader("Access-Control-Allow-Origin", "https://{USERNAME}.github.io")
```

## 8. フロントエンド実装仕様

### ファイル構成

```
index.html      — 入力フォームページ
stats.html      — 統計ダッシュボードページ
quiz.html       — クイズページ
css/style.css   — 共通スタイル
js/config.js    — GASエンドポイントURL等の設定
js/form.js      — 入力フォームのロジック
js/suggest.js   — 類似銘柄サジェストのロジック
js/stats.js     — 統計表示のロジック
js/quiz.js      — クイズのロジック
```

### 類似銘柄サジェストの実装

- Levenshtein距離の計算はJavaScriptで実装（外部ライブラリ不要）
- 銘柄一覧はセッション中にメモリキャッシュ（ページ読み込みごとに1回だけGAS呼び出し）

### グラフライブラリ

- **Chart.js**（CDN経由）を使用
- 追加費用なし

## 9. セキュリティ

| 脅威 | 対策 |
|---|---|
| 第三者からの不正アクセス | GASのアクセス設定を「自分のみ」に設定 |
| CSRF | GASのトークン検証（GAS標準） |
| XSS | フロントエンドでの入力サニタイズ |
| データ漏洩 | スプレッドシートの共有設定を「非公開」に設定 |

---

## 10. 開発・デプロイ手順（概要）

1. Googleスプレッドシートを作成し、スプレッドシートIDを控える
2. GASプロジェクトを作成し、スプレッドシートIDを設定
3. GASをWebアプリとしてデプロイ（アクセス: 自分のみ）
4. デプロイURLをフロントエンドの `js/config.js` に設定
5. フロントエンドをGitHubリポジトリにプッシュ
6. GitHub PagesをリポジトリのSettingsから有効化
